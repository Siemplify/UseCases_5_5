Siemplify 5.5.0 Is Here - So Whats New?

Our latest release was developed with 100% customer feedback in mind and a pinch of innovation.
We focused on improving the Playbook Lifecycle Management, Integrations Management, introducing Smarter SOC Notifications and Environments and created a new Marketplace for Use Cases that lets you share and download use cases developed by Siemplify power users and the Siemplify Community.

Playbook LifeCycle Management

  • So, how do you know how effective your Playbook is? This new set of features brings more clarity to the world of Playbooks.

    From now on, you can view at a glance how efficient the Playbook is in the new Playbook Monitoring panel, and using the various charts, figure out how to tweak the Playbook to achieve optimal performance.

    In addition, the Playbook Summary in the Cases screen allows you to immediately hone in on an error or a manual action, thereby saving you vital work time.

  • We have also provided an out of the box Playbooks Dashboard providing management with even greater visibility.

New Integration Marketplace

  • With Siemplify release 5.5.0, security analysts can now configure several instances of a single Integration for one environment. This feature provides both MSSPs and enterprises with the ability to choose this instance within a Playbook thereby allowing the Playbook to run on a specific configuration for a specific environment.

Use Cases Marketplace

  • This free marketplace provides a library of use cases developed by Siemplify and other power users. Each use case is an implementation of a solution - such as Malware Analysis or a Phishing Automation.

    Each use case is deployed with the test data, playbooks, connectors, integrations, settings and guides required for an end-to-end flow. Your Siemplify experience is boosted with every use case you deploy

Notifications

  • The notifications mechanism has been greatly enhanced. Siemplify now pushes user notifications both to the Siemplify platform and to your email address for a variety of different situations. All of the User Notifications can be customized according to your needs.

    In addition, the user notifications often contain links to cases enabling you to locate what you need quickly and easily. This feature also enables you to act with a much quicker response time to case actions or Playbook manual actions, among others.

Environments

  • The concept of Environments has been reworked slightly to allow MSSPs and Enterprise customers to benefit from the ability to select multiple tenancies for system configurations for single functions.

    In addition, the no environment that was provided out of the box, has been renamed to default environment and can now be treated like any other environment with its own set of permissions.
